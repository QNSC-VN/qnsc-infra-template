# Security Policy

## Reporting a Vulnerability

**Do not open a public GitHub issue for security vulnerabilities.**

Email: **security@qnsc.vn**

Acknowledgement within **48 hours**, status update within **7 days**.

## Scope

These are shared infrastructure modules consumed by every QNSC product. In scope:

- Insecure defaults (e.g. public S3 buckets, permissive security groups, missing encryption)
- Over-permissive IAM generated by a module
- Missing or weak TLS / WAF configuration
- Any module change that would weaken a consuming product's security posture

## Practices

- Modules default to least-privilege and encryption-on.
- Changes are reviewed by CODEOWNERS before merge.
- CI runs `tofu validate` and `tflint` on every change.
